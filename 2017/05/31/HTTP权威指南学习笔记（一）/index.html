HTTP权威指南学习笔记（一）

就是对《http权威指南》学习的一点点记录。

报文

分为请求报文和响应报文，包含起始行、首部字段、主体

连接

tcp/ip

tcp/ip是因特网传输协议

tcp:

  • 无差错数据传输
  • 按序传输
  • 未分段的数据流（在任意时刻以任意尺寸将数据发送出去）
    http     应用层
tcp     数据层
ip      网络层
链路接口  数据链路层
网络硬件  物理层

ip: 国际协议地址和端口号在客户端和服务器之间建立一条连接

使用url获取http服务器的ip地址和端口号

使用telnet程序

web服务器会将telnet程序作为一个web客户端来处理

其实就相当于手动输入报文

同时也可以用netcat(nc)

web的结构组建

  1. 代理
  2. 缓存
  3. 网关（连接其他应用程序的特殊web服务器）
  4. 隧道（对http通信报文进行盲转发的特殊代理）
  5. agent代理（发起自动http请求的半智能web客户端

URL

URL

url: 方案://服务器位置/路径
告知浏览器如何对资源进行访问和处理。定义了用户所需的特定资源，它位于何处以及如何获取它

语法：

://:@:/;?#
